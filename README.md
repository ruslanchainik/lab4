## 1. Задание

1. Построить односвязную (без скрытых слоёв) сеть для бинарной классификации с активациями `BReLU` и `Sigmoid` без использования фреймворков.
2. Реализовать SGD с регулируемым размером батча и модификацию Adam (Nesterov momentum + RMSProp).
3. Сгенерировать два датасета (`make_moons`, `make_classification`) с `random_state = 408607`.
4. Обучить модель из п.1 с функцией потерь `Cross-entropy`, разделение 60/20/20.
5. Реализовать перцептрон с одним скрытым слоем (многослойную сеть) и повторить п.4.
6. Сформировать отчёт.

Состав активаций и функция потерь в п.5 совпадают с п.4 — добавлен именно скрытый слой.

## 4. Реализация

### 4.1 Активации

* `BReLU(x) = clip(x, 0, 1)`. Производная равна `1` на `(0, 1)` и `0` вне.
* `Sigmoid(x)` реализована численно устойчиво (раздельная обработка положительных и отрицательных значений аргумента).

### 4.2 Архитектуры

* **`SingleLayerNet` (п.4):** `Linear(n_in → 1) → BReLU → Sigmoid`.
* **`Perceptron` (п.5):** `Linear(n_in → h) → BReLU → Linear(h → 1) → Sigmoid`, `h ∈ {4, 8, 16, 32}`.

Веса инициализируются по методу He (`scale = sqrt(2 / n_in)`).

### 4.3 Функция потерь

Бинарная кросс-энтропия со стабилизацией через `clip` вероятностей в `[eps, 1 − eps]`. Градиент по предсигмоидному значению равен `(ŷ − y)`.

### 4.4 Оптимизатор

Реализован Nadam-вариант (Nesterov momentum + RMSProp):

* экспоненциальные средние первого и второго моментов с коэффициентами `β1 = 0.9`, `β2 = 0.999`;
* коррекция смещений `m̂`, `v̂`;
* шаг с Nesterov-видом числителя: `β1 · m̂ + (1 − β1) · g / (1 − β1^t)`;
* нормировка на `sqrt(v̂) + ε`.

Размер батча задаётся параметром и используется в `iterate_minibatches`.

### 4.5 Подбор модели

Поиск по сетке гиперпараметров (`lr × batch_size [× hidden]`) с критерием минимума валидационной кросс-энтропии. Применяется ранняя остановка по `val_loss` с восстановлением лучших весов (`patience = 60`). Дополнительно подбирается порог классификации по максимуму F1 на валидационной выборке: выход одноуровневой сети после `Sigmoid(BReLU(·))` сосредоточен в `[0.5; 0.731]`, и порог `0.5` неприменим без коррекции.

## 8. Сравнительный анализ

* На `make_moons` (нелинейно разделимая задача) односвязная сеть достигает только `Acc ≈ 0.79` и `ROC-AUC ≈ 0.80`, поскольку одно линейное преобразование не способно описать форму классов. Перцептрон со скрытым слоем (`h = 8`) достигает `Acc = 0.975` и `ROC-AUC = 0.994` — добавление скрытого слоя является принципиальным условием решения задачи.
* На `make_classification` задача почти линейно разделима в подпространстве информативных признаков. Односвязная сеть достигает `Acc = 0.975` (`ROC-AUC = 0.975`); перцептрон сохраняет ту же accuracy и достигает `ROC-AUC = 1.0` при существенно меньшей тестовой кросс-энтропии (`0.108` против `0.528`).
* Тестовая cross-entropy у односвязной модели заметно выше из-за ограниченности выхода `Sigmoid(BReLU(·)) ∈ [0.5; 0.731]`: вероятности плохо калиброваны, поэтому критически важно подбирать порог классификации по валидационной выборке.
* Адаптация оптимизатора (Nesterov + RMSProp) обеспечивает сходимость менее чем за 200 эпох на обоих датасетах при `lr ∈ [0.01; 0.1]`. Уменьшение `batch_size` стабильно ускоряет начальное снижение валидационной потери, но повышает её дисперсию.
